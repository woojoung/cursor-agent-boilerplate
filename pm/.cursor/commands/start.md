# 시작 (역질문 모드) — 기획/PM

사용자가 **뭘 해야 할지 모를 때** 옵션을 제시하고, 선택에 따라 해당 워크플로우를 안내·실행한다.

## 실행 조건

- 사용자가 **`/start`** 를 실행했을 때.
- 또는 "도와줘", "뭐 할 수 있어?", "기획 관련해서", "PM 쪽 일" 등 **작업 내용이 불명확한 요청**을 했을 때.

## 실행 절차

1. **옵션 제시**  
   이 포지션의 **PROMPT_TEMPLATES.md**에 있는 "에이전트가 역질문할 때 제시할 옵션" 표를 참고하여, 아래를 사용자에게 보여 준다.
   - ① 스펙(PRD) 초안 작성
   - ② 태스크 분해
   - ③ 리스크·의존성 점검
   - ④ 이슈 템플릿 채우기  
   → "위 중 어떤 작업을 도와드릴까요? 번호나 작업명으로 알려주세요."

2. **선택에 따른 추가 질문**  
   사용자가 선택한 번호(또는 작업명)에 따라, PROMPT_TEMPLATES.md의 "역질문 플로우"에 적힌 **최소한의 추가 질문**을 한두 개 한다.

3. **워크플로우 실행·안내**  
   답을 받은 뒤 해당 명령을 실행하거나 안내한다.
   - 1 → `/spec-template` 실행 또는 스펙 초안 생성
   - 2 → `/task-breakdown` 실행 또는 태스크 분해
   - 3 → `/risk-check` 실행 또는 리스크 정리
   - 4 → 이슈 본문 초안 생성

## 주의

- 사용자가 이미 구체적으로 "스펙 초안 만들어줘"라고 했으면 `/start` 대신 해당 명령(spec-template)으로 바로 진행한다.
- 역질문은 **포지션을 잘 모르는 사용자**가 "뭐 할 수 있어?"라고 할 때만 옵션을 보여 주면 된다.

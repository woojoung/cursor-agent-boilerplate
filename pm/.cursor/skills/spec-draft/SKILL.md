# Spec Draft Skill (pm)

## 초안 구조

1. **개요**: 무엇을·왜(2–3문장). 비즈니스 임팩트.
2. **문제**: 현재 페인·정량 근거.
3. **목표·성공 지표**: 측정 가능한 기준.
4. **요구사항**: 기능(P0/P1/P2)·비기능. AC로 검증 가능하게.
5. **제약·엣지 케이스**: 예외·중간 이탈·에러·중복 시나리오.
6. **참고**: 링크·담당.

## 원칙

- "왜" 먼저, "무엇" 다음. 구현 세부는 팀 협의.
- 엣지 케이스를 명시해 요구 누락·오해 방지.

# Cursor 대화창에서 쓸 수 있는 프롬프트 모음

> 이 레포(`cursor-agent-boilerplate`)는 **포지션별 폴더**(backend, frontend-web, frontend-mobile, qa, infra, pm, design)로 구성되어 있습니다.  
> **작업할 포지션 폴더를 Cursor에서 Open** 한 뒤, 아래 문장을 **채팅 입력창에 복사해 붙여 넣어** 사용하면 됩니다. 슬래시(`/`) 대신 문장으로 요청할 때 참고하세요.  
> **포지션을 잘 모르는 경우**: 각 포지션 폴더에서 **`/start`** 를 입력하면, 에이전트가 **자주 하는 작업 옵션**을 보여 주고 **역으로 질문**해 줍니다. 선택만 하면 해당 작업을 진행할 수 있습니다. 각 포지션의 **PROMPT_TEMPLATES.md**에 옵션과 "자주 쓰는 프롬프트"가 정리되어 있습니다.

## 참고: 포지션별 적용 · Slack 연동 · 역할 정의 레포 (맥락)

아래는 **포지션** 적용 요약(프론트는 웹·모바일 구분), **역할 정의 레포 vs 실무 레포**, **Slack 봇 연결** 흐름입니다. 대화 시 "아래 참고 내용 기준으로"라고 하면 이 맥락을 함께 사용할 수 있습니다.

### 포지션별로 쓸 수 있는 구조 (이 레포는 이미 이 구조로 되어 있음)

- **백엔드** → `backend/`. `.cursor/`·AGENTS.md·플러그인 전체 구현. 실무 레포에 복사 또는 플러그인으로 적용.
- **프론트엔드(웹·Next.js)** → `frontend-web/`. 최소 `.cursor/rules/main.mdc` + AGENTS.md. 확장 시 Next.js·접근성·테스트 규칙/명령/스킬/에이전트 추가.
- **프론트엔드(모바일·Flutter)** → `frontend-mobile/`. 최소 규칙·AGENTS.md. 확장 시 Flutter/Dart·위젯·플랫폼 규칙/명령/스킬/에이전트 추가.
- **QA/테스트** → `qa/`. 최소 규칙·AGENTS.md. 확장 시 테스트·시나리오·자동화 Rules, 시나리오 도출·회귀 범위·버그 템플릿 Command/Skill/Agent 추가.
- **인프라/DevOps** → `infra/`. 최소 규칙·AGENTS.md. 확장 시 IaC·배포·보안 Rules, 파이프라인·장애 대응 Command/Skill/Agent 추가.
- **기획/PM** → `pm/`. 최소 규칙·AGENTS.md. 확장 시 스펙·이슈 Rules, 스펙 초안·태스크 분해·리스크 체크 Command/Skill/Agent 추가.
- **디자인/브랜딩** → `design/`. 최소 규칙·AGENTS.md. 확장 시 디자인 시스템·명명 Rules, 컴포넌트 규격·브랜드·접근성 검토 Command/Skill/Agent 추가.

**사용 시**: Cursor에서 **작업할 포지션 폴더**(예: `backend/`, `frontend-web/`)를 Open 하면 해당 포지션의 규칙·AGENTS.md가 적용됩니다.  
**실무 레포 적용·Slack 봇 연동**: 각 포지션 폴더에는 `.cursor-plugin/`이 있어 플러그인 소스로 사용 가능합니다. 실무 레포에서의 플러그인/복사 방법과 Slack 봇과의 팀 워크플로우는 **[INTEGRATION.md](INTEGRATION.md)** 를 참고하세요.

### 목표 정리 (Slack · BOSS · Cursor)

- **포지션**: 백엔드, 프론트엔드(웹·Next.js), 프론트엔드(모바일·Flutter), QA/테스트, 인프라/DevOps, 기획/PM, 디자인/브랜딩 → 역할별 **히스토리·문서화 필수**
- **역할별 에이전트·규칙**이 정의된 **레포(들)** 존재
- **Slack 연동**: 포지션별(또는 역할 전환) **슬랙 봇**과 대화. **BOSS**는 슬랙에서 관리·의사결정. 실무는 **포지션별 레포**에서 진행하고 결과를 **슬랙으로 공유**. **Cursor IDE**는 각 포지션이 실무를 효율적으로 할 수 있게 사용.

### 큰 흐름 (역할 정의 레포 · 실무 레포 · Slack)

- **역할 정의 레포 1개**: 안에 `backend/`, `frontend-web/`(Next.js), `frontend-mobile/`(Flutter), `qa/`, `infra/`, `pm/`, `design/` 등 포지션별로 Rules, Commands, Skills, Agents, AGENTS.md 정의. 히스토리·문서화 규칙 포함.
- **실무 작업 레포**: 팀/프로젝트/포지션마다 따로 둠 (예: product-a-backend, product-a-frontend). 역할 정의를 **플러그인처럼** 실무 레포에 적용 (`.cursor/` + AGENTS.md 복사 또는 플러그인 참조).
- **Slack 봇 연결**: 봇은 **역할 정의 레포 1개만** 규칙/프롬프트 소스로 연결. 실무 레포는 Git 웹훅 등으로 이벤트만 구독해 작업 현황·알림용으로 사용.
- **실무 레포 전략**: 역할 정의는 1레포 유지. 실무는 **포지션/프로젝트 단위로 여러 레포**로 나누는 쪽 권장 (권한·CI/CD·담당자 인지·슬랙 매핑이 단순해짐).

### 한 줄 요약

- **역할 정의 레포**: 포지션별 Rules·Agents·문서화 규칙. 슬랙 봇이 "역할 정의"를 여기서 로드.
- **실무**: 포지션별 실무 레포(또는 모노레포의 포지션 폴더). Cursor는 여기서 해당 포지션 설정으로 작업.
- **Slack**: 포지션별 채널/봇 대화 + BOSS 채널에서 의사결정·승인.
- **히스토리·문서화**: 역할 정의 레포의 Rules/AGENTS.md에 "문서·커밋·PR 규칙"으로 넣고, Cursor와 봇이 그 규칙을 따르게 하면 "역할별 히스토리 및 문서화 필수"를 만족.

## 사용 방법

1. **Cursor** 실행 후 **File → Open Folder**에서 **작업할 포지션 폴더**를 엽니다.  
   - 예: `cursor-agent-boilerplate/backend` (백엔드), `cursor-agent-boilerplate/frontend-web` (프론트 웹)  
   - 루트(`cursor-agent-boilerplate`)가 아닌 **포지션 폴더**를 열어야 해당 규칙·명령이 적용됩니다.
2. **채팅**을 엽니다 (`Cmd + L` 또는 채팅 아이콘).
3. 위 프롬프트 중 하나를 **복사해 채팅 입력창에 붙여 넣고** 전송합니다.
4. 필요하면 "현재 폴더만", "backend 기준으로"처럼 범위를 한 줄로 추가해 주세요.

슬래시 명령(`/plan-from-spec`, `/review-code` 등)은 **backend** 폴더를 연 상태에서 사용할 수 있습니다. 다른 포지션은 확장 시 해당 폴더에 commands를 추가하면 됩니다.

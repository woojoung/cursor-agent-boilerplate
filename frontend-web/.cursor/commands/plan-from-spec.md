# Plan from Spec (frontend-web)

spec.md를 분석하여 구현 계획(plan.md)을 생성한다. 컴포넌트·라우트·테스트·접근성 단계를 포함한다.

## 실행 절차

1. spec.md 탐색 및 분석(목표, 요구사항, 제약·접근성).
2. app/components/lib 구조·유사 기능 참고.
3. plan.md 생성: Phase별로 라우트·레이아웃 → 컴포넌트 설계 → 구현 → 테스트·접근성 검증.
4. 사용자 승인을 기다린다.

## plan.md 포함 항목

- Phase 1: 라우트·레이아웃·데이터 요구 정리
- Phase 2: 컴포넌트 분할·props·상태 설계
- Phase 3: 구현 (Server/Client 구분, fetch·캐시)
- Phase 4: 단위·컴포넌트 테스트, 접근성 점검

완료 시: "plan.md를 검토하고 승인해 주세요."라고 안내한다.

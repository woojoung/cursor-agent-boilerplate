# MR 설명 초안 (frontend-web)

staged 변경분 기준으로 커밋 메시지·MR(Merge Request) 설명 초안을 생성한다.

## 실행 절차

1. staged(또는 최근 커밋) 변경 파일·diff 확인.
2. 변경 목적·영향 범위(라우트/컴포넌트/훅/스타일) 요약.
3. 커밋 메시지 초안 (Conventional Commits 권장: feat/fix/docs 등).
4. MR 제목·설명 초안 (변경 요약, 테스트·접근성 체크 여부, 스크린샷 필요 시 명시).

완료 시: "위 내용을 GitLab MR에 붙여 넣고 리뷰어 체크리스트를 보완하세요."라고 안내한다.

# Scenario Writing Skill (QA)

## 목적

- 스펙·AC(수용 조건)를 실행 가능한 테스트 케이스로 변환
- 정상·경계·예외·엣지 시나리오 도출
- 전제조건·단계·기대결과를 명확히 기술

## 절차

1. 요구사항/AC 목록 추출
2. 각 요구당 정상 시나리오 1개 이상
3. 입력 경계(빈값·최대·특수문자)·에러 처리·권한 시나리오 추가
4. ID·제목·전제·단계·데이터·기대결과 형식으로 작성

## 품질

- 한 케이스당 하나의 검증 목표. 기대 결과는 "~해야 한다" 형태로 측정 가능하게.

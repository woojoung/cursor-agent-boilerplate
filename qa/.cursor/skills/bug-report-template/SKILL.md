# Bug Report Template Skill (QA)

## 리포트 구성

- **제목**: 한 문장으로 현상 요약
- **환경**: OS·브라우저/앱 버전·계정·데이터
- **전제조건**: 재현 전 상태
- **재현 단계**: 번호별로 누구나 따라 할 수 있게
- **기대 결과**: 명시적 기대
- **실제 결과**: 스크린샷·에러 메시지·로그
- **심각도/우선순위**: 팀 정책에 맞게 (Critical/Major/Minor 등)
- **추가 정보**: 로그·네트워크 캡처·동영상

## 원칙

- 재현 가능하도록 단계·데이터 구체적으로. "가끔" 발생 시 재현율·패턴 기술.

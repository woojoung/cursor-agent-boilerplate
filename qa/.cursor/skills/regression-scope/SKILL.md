# Regression Scope Skill (QA)

## 목적

- 코드/기능 변경 시 영향받는 영역 식별
- 회귀 테스트 범위·우선순위 제안
- 자동화 스위트와 매핑해 실행할 케이스 선정

## 절차

1. 변경 사항(커밋·MR·기능 목록) 파악
2. 영향 모듈·화면·API·플로우 도출
3. 기존 테스트 케이스/자동화와 매핑
4. P0(필수)·P1(권장)·P2(선택) 우선순위 부여
5. 수동 vs 자동 실행 권장 정리

## 산출

- 영향 범위 요약, 실행할 테스트 목록(또는 ID), 예상 소요 시간.

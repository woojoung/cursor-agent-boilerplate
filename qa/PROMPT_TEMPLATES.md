# QA 포지션 — 자주 쓰는 요청·역질문 옵션

> **`/start`** 또는 "QA 도와줘", "뭐 할 수 있어?"라고 하면, 아래 옵션을 보여 주고 선택을 받은 뒤 해당 워크플로우를 진행합니다.

---

## 에이전트가 역질문할 때 제시할 옵션

| 번호 | 작업 | 사용자가 선택 시 에이전트가 할 일 | 연결 명령/스킬 |
|------|------|-----------------------------------|----------------|
| **1** | **스펙→테스트 시나리오 도출** | 스펙/요구사항 문서를 분석해 테스트 시나리오·케이스 초안 생성. 정상·경계·예외 시나리오 포함. | `/scenario-from-spec`, scenario-writing 스킬 |
| **2** | **테스트 계획 수립** | 변경 범위·릴리스 기준 회귀 범위·우선순위(P0/P1/P2)·수동/자동·일정 초안. | `/test-plan`, regression-scope 스킬 |
| **3** | **버그 리포트 템플릿** | 버그 발견 시 채울 리포트 템플릿 생성. 환경·재현 단계·기대/실제·심각도. | `/bug-template`, bug-report-template 스킬 |

---

## 역질문 플로우

1. 위 옵션 1~3 제시 → "몇 번을 도와드릴까요?"
2. 1번: "스펙/요구사항 문서 경로나 내용을 알려주세요."
3. 2번: "어떤 변경(MR·커밋·기능) 기준으로 테스트 계획을 세울까요?"
4. 3번: "버그 현상·환경을 한두 줄로 알려주시면, 그에 맞는 리포트 초안을 만들어 드립니다."
5. 해당 명령 실행 또는 안내.

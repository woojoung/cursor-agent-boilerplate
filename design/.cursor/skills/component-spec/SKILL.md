# Component Spec Skill (design)

## 규격 구성

- **이름·목적**: 컴포넌트 역할·사용 맥락.
- **변형**: variant, size, state. 각각 예시.
- **토큰 매핑**: 색·간격·타이포·보더 등 사용 토큰 목록.
- **접근성**: 대비·포커스·키보드·스크린리더 요구사항.
- **Do/Don't**: 권장·비권장 사용 예.
- **참고**: 피그마·코드 링크.

## 원칙

- 개발·QA가 구현·검증할 수 있도록 구체적으로. 토큰 이름은 디자인 시스템과 동일.

# GitLab Workflow Skill

GitLab을 활용한 커밋, 푸시, MR 자동화 워크플로우입니다.

## 브랜치 전략

### 브랜치 구조

```
{ROOT_BRANCH} (설정에 따라 다름)
  │
  ├── feature/{gitlab_account}/{feature-name}
  │     예: feature/john.doe/user-auth
  │
  ├── bugfix/{gitlab_account}/{bug-name}
  │     예: bugfix/jane.kim/login-error
  │
  └── hotfix/{version}/{fix-name}
        예: hotfix/1.2.3/security-patch

※ ROOT_BRANCH는 commit-push 스킬의 {ROOT_BRANCH} 값을 사용합니다.
```

### 브랜치 네이밍 규칙

```
{type}/{gitlab_account}/{description}

type:
- feature  : 새 기능 개발
- bugfix   : 버그 수정
- hotfix   : 긴급 수정 (프로덕션)
- refactor : 리팩토링
- test     : 테스트 추가/수정

description:
- 소문자, 하이픈(-) 구분
- 간결하고 명확하게
- 예: user-authentication, fix-login-error
```

## 커밋 메시지 컨벤션

### 형식

```
{type}: {subject}

{body}

{footer}
```

### Type 종류

| Type       | 설명     | 예시                           |
|------------|--------|------------------------------|
| `feat`     | 새로운 기능 | feat: 사용자 로그인 기능 추가          |
| `fix`      | 버그 수정  | fix: 토큰 만료 처리 오류 수정          |
| `refactor` | 리팩토링   | refactor: AuthService 메서드 분리 |
| `test`     | 테스트    | test: AuthServiceTest 추가     |
| `docs`     | 문서     | docs: API 문서 업데이트            |
| `style`    | 포맷팅    | style: 코드 포맷 정리              |
| `chore`    | 빌드/설정  | chore: gradle 버전 업데이트        |
| `perf`     | 성능 개선  | perf: 쿼리 최적화                 |

### 좋은 커밋 메시지 예시

```
feat: JWT 기반 사용자 인증 구현

- 로그인 API 추가 (POST /api/auth/login)
- 토큰 발급 및 검증 로직 구현
- Refresh Token 지원

Closes: JIRA-1234
```

```
fix: 로그인 시 비밀번호 검증 오류 수정

BCrypt 인코딩된 비밀번호와 원본 비밀번호 비교 순서가
잘못되어 있던 문제를 수정함

Fixes: JIRA-5678
```

### 나쁜 커밋 메시지

```
❌ 수정
❌ 버그 fix
❌ 작업중
❌ asdf
❌ WIP
```

## Git 명령어 플로우

### 기능 개발 시작

```bash
# 1. 루트 브랜치 최신화 (ROOT_BRANCH: 설정된 루트 브랜치)
git checkout {ROOT_BRANCH}
git pull origin {ROOT_BRANCH}

# 2. 피처 브랜치 생성
git checkout -b feature/john.doe/user-auth

# 3. 작업 후 커밋
git add src/main/java/com/example/auth/
git commit -m "feat: 사용자 인증 기능 구현"

# 4. 푸시
git push -u origin feature/john.doe/user-auth

# 5. MR 생성 (GitLab MCP 또는 CLI)
```

### MR 머지 후

```bash
# 1. 루트 브랜치로 이동
git checkout {ROOT_BRANCH}

# 2. 최신화
git pull origin {ROOT_BRANCH}
```

## GitLab MCP 연동

### MR 생성 API

```javascript
// GitLab API 호출 (MCP 통해)
POST / projects / {project_id}
/merge_requests

{
  "source_branch"
:
  "feature/john.doe/user-auth",
    "target_branch"
:
  "{ROOT_BRANCH}",  // 설정된 루트 브랜치
    "title"
:
  "feat: 사용자 인증 기능 구현",
    "description"
:
  "## Summary\n...",
    "remove_source_branch"
:
  true,
    "squash"
:
  false
}
```

### 설정

```json
{
  "gitlab-workflow": {
    "rootBranch": "{ROOT_BRANCH}",
    "branchPrefix": "feature/{gitlab_account}",
    "removeSourceBranch": true,
    "squashOnMerge": false,
    "autoAssignReviewer": true,
    "defaultReviewers": [
      "@tech-lead"
    ]
  }
}
```

## 연관 명령어

- `/mr-gitlab`: 커밋/푸시/MR 자동화 (전체 워크플로우)
- `/commit-push`: 커밋 및 푸시만 실행 (MR 생성 후 추가 수정 시 사용)
- `/review-code`: MR 생성 전 코드 리뷰

## 명령어 사용 시나리오

```
# 시나리오 1: 새 MR 생성
/mr-gitlab feat: 사용자 인증 구현
→ commit → push → review → MR 생성

# 시나리오 2: MR 피드백 반영 후 추가 수정
/commit-push fix: 코드 리뷰 피드백 반영
→ commit → push (기존 MR에 자동 반영)

# 시나리오 3: 단순 백업 푸시
/commit-push chore: WIP 저장
→ commit → push
```

## 연관 에이전트

- **code-reviewer**: MR 생성 시 자동 리뷰
- **security-reviewer**: 보안 검토

# 백엔드 포지션 — 자주 쓰는 요청·역질문 옵션

> **`/start`** 또는 "백엔드 도와줘", "뭐 할 수 있어?"라고 하면, 아래 옵션을 보여 주고 선택을 받은 뒤 해당 워크플로우를 진행합니다.

---

## 에이전트가 역질문할 때 제시할 옵션

| 번호 | 작업 | 사용자가 선택 시 에이전트가 할 일 | 연결 명령/스킬 |
|------|------|-----------------------------------|----------------|
| **1** | **스펙→계획→태스크** | spec.md 분석 후 plan.md·task.md 생성. 승인 후 task 단위 구현. | `/spec-template`, `/plan-from-spec`, plan.md·task.md |
| **2** | **코드 리뷰** | MR/변경 범위 기준으로 리뷰 체크리스트 적용. 보안·테스트·설계 관점 제안. | `/review-code`, code-reviewer 에이전트 |
| **3** | **MR 초안** | 변경 내용 요약·체크리스트·테스트 방법으로 MR 설명 초안 생성. | `/mr-gitlab`, mr-gitlab 스킬 |
| **4** | **TDD·테스트** | 요구사항/메서드 기준으로 테스트 케이스·단위 테스트 초안 작성. | TDD 규칙, test-writer 에이전트 |
| **5** | **새 레포 세팅** | Java/Spring Boot 새 프로젝트 생성 후 이 포지션 .cursor·AGENTS.md 복사. | `/setup-repo` |

---

## 역질문 플로우

1. 옵션 1~5 제시 → "몇 번을 도와드릴까요?"
2. 1번: "스펙 문서(spec.md) 경로나 요구사항 요약을 알려주세요."
3. 2번: "리뷰할 브랜치/MR 또는 변경된 파일 경로를 알려주세요."
4. 3번: "변경한 기능/파일을 한 줄로 알려주시면 MR 설명 초안을 만들겠습니다."
5. 4번: "어떤 클래스/메서드에 대한 테스트인가요?"
6. 5번: "프로젝트 이름(폴더명)을 알려주세요."
7. 해당 명령 실행 또는 안내.
